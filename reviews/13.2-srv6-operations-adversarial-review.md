## CRITICAL Findings

### C1: uSID Count Per Carrier Is Wrong (4 claimed, actually 6)
- **Location:** uSID section
- **Issue:** Claims "4 micro-SIDs" per carrier. F3216 format actually fits **6** uSIDs (32-bit block + 6×16-bit slots = 128 bits)
- **Impact:** Would be marked wrong on CCIE written. Understates the compression benefit.
- **Fix:** Replace "4 micro-SIDs" with "6 micro-SIDs (in F3216 format)"

### C2: All Examples Use ULA After Saying "Don't Use ULAs"
- **Location:** SID Allocation Strategy
- **Issue:** Guide warns against ULAs for production, then uses `fc00:0::/32` (ULA space) in every example
- **Impact:** Self-contradictory. RFC 9602 allocated `5f00::/16` for SRv6 SIDs — examples should use that
- **Fix:** Use `5f00::` prefixes or clearly label fc00:: as "lab/documentation only"

### C3: uSID Block Convention Claim Is Factually Wrong
- **Location:** uSID Block Management
- **Issue:** Claims "`fc00::/16` is the industry convention for uSID block" — it's not. RFC 9602 → `5f00::/16`. Cisco labs use `fcbb:bb00::/32`.
- **Fix:** Reference RFC 9602 properly

### C4: uSID Locator Prefix Size Is Wrong in Config
- **Location:** uSID Block Management, IOS-XR config
- **Issue:** Shows `/32` as "per-node uSID locator" — actually `/32` is the shared block, per-node is `/48`. Also says "/32 is narrower than /48" which is backwards.
- **Fix:** Use `/48` per-node locator per Cisco docs

## HIGH Findings

### H1: H.Encaps.Red Omitted from MTU Math
- Overhead table assumes standard H.Encaps. Modern IOS-XR/Junos default to H.Encaps.Red (first segment in DA only, not duplicated in SRH)
- 3-segment path: 80 bytes with H.Encaps.Red, not 96 as stated

### H2: `clear isis flex-algo 128 topology` — Likely Fictional Command
- Not in any IOS-XR command reference. Could confuse during troubleshooting or cause unexpected reconvergence.

### H3: PSP/USP/USD Flavors Completely Omitted
- Fundamental to SRv6 forwarding, tested on CCIE SP Written, visible in all SID outputs. Mentioned once in config but never explained.

### H4: Title Claims "Scale" But Doesn't Address Scale
- Missing: TCAM/FIB limits, IS-IS flooding at 1000+ nodes, PCE scalability, SID churn during upgrades, per-platform Max-SL limits, telemetry at scale

### H5: NCS5500 hw-module Profile Requirement Not Mentioned
- `hw-module profile segment-routing srv6` (requires LC reload) — day-zero gotcha completely absent
