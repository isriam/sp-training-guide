# Peer Review — Section 13.2: SRv6 Operations at Scale
**Date:** 2026-03-26

## Summary
Strong writing and good operational mindset, but enough factual errors and fabricated CLIs to mislead a CCIE/JNCIE candidate on 4-5 exam questions.

---

## Issues Found

### CRITICAL

**1. ULA Contradiction** — Same as adversarial C2. All examples use ULA after warning against it. Peer reviewer adds nuance: Cisco actually *recommends* ULA for intra-domain SRv6 (SoftBank, Iliad/Free use it). The guide's blanket "don't use ULA" advice is itself debatable.

**2. Fabricated IOS-XR Packet Capture CLI** — `monitor capture` is IOS-XE only. Does NOT exist on IOS-XR (ASR9K, NCS5500, 8000). Should use SPAN/monitor-session or shell-based tcpdump.

### WRONG CLI COMMANDS

**3. Junos SR-TE show commands wrong** — `show spring-te lsp` doesn't exist. Correct: `show spring-traffic-engineering lsp`

**4. IOS-XR TCP MSS Adjust wrong** — `ipv4 tcp-mss-adjust` is not valid IOS-XR. ASR9K uses `hw-module location` syntax. NCS5500/8000 varies.

**5. `clear isis flex-algo 128 topology`** — Same as adversarial H2. Likely fabricated.

**6. Junos `show route table srv6.sid.0`** — Same as adversarial M5. Table name unverifiable.

**7. Junos SR-TE config hierarchy** — `routing-options segment-routing-te` likely wrong. Should be `protocols source-packet-routing` or `protocols spring-traffic-engineering` depending on version.

### TECHNICAL ERRORS

**8. Locator table /64 entry** — Claims "0 bits for function" which is nonsensical. Same issue as adversarial M1 — bit math doesn't add to 128.

**9. MTU overhead table** — Same as adversarial H1. Missing H.Encaps.Red distinction. 1-segment with reduced encap = 40 bytes, not 64.

**10. tcpdump filter limitation** — Same as adversarial M3.

### MISSING CONTENT (NEW findings not in adversarial review)

**11. H.Encaps/H.Insert/Reduced variants completely absent** — Four headend behaviors never explained. Affects MTU, security, troubleshooting.

**12. PSP/USP/USD flavors** — Same as adversarial H3.

**13. `monitor capture` is IOS-XE, not IOS-XR** — This is a NEW critical finding the adversarial review missed. The entire NCS5500 packet capture section uses wrong platform CLI.
